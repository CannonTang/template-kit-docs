动画模块

无需 Animator Controller，用更简单的方式驱动动画

模块包含

  • 简易动画器 — 快速、一键、无代码地生成物体或 UI 的简单动画
  • 路径动画器 — 快速、一键、无代码地生成物体或 UI 的路径动画
  • 帧动画控制器 — 控制 Legacy Animation 组件的播放帧段

创建方式

右键 Hierarchy > GameObject > 模板 > 动画 > [选择组件]

简易动画器

SimpleAnimator — 一键生成基础动画

它能做什么

快速、一键、无代码地生成物体或 UI 的简单动画，让对象在起点和终点之间自动过渡。支持两种模式：

  • Transform 模式 — 位移、旋转、缩放（包括 RectTransform 的 anchoredPosition、sizeDelta 等）
  • ImageFill 模式 — 控制 Image 的 fillAmount（进度条、冷却环等）

更适合做轻量、直接、无需 Animator Controller 的交互动画：面板滑入、按钮弹出、图标旋转、进度填充、冷却圈推进这类两端状态明确的场景都很合适。

使用步骤

1

右键创建：GameObject > 模板 > 动画 > 简易动画器 (位移旋转缩放)

或选择 简易动画器 (图片填充) 进入 ImageFill 模式。

2

在 Inspector 中点击 QuickInit 按钮，会自动生成两个子物体作为标记：

  • OriginMarker — 代表动画起点的 Transform 状态
  • DisplayMarker — 代表动画终点的 Transform 状态
3

在 Scene 视图中移动 OriginMarker 和 DisplayMarker 到想要的位置/旋转/缩放。

4

跳转到下方获取完整 API，或进入调试标签页直接预览测试效果。

UI 面板位移演示

更偏向界面交互场景，例如抽屉面板、设置框、提示条的滑入滑出。推荐搭配 Use Local Space 使用。

ImageFill 填充演示

适合血条、冷却环、加载进度。只关心起点和终点的 fillAmount，不需要单独写数值补间逻辑。

演示说明

上方主视频现在以 3D / 通用 Transform 为主，突出 SimpleAnimator 最基础也最通用的两点补间能力；下方两个补充演示再分别覆盖 UI 面板位移 和 ImageFill，版面层级会更清楚。

主要属性

属性说明
ModeTransform 或 ImageFill
Duration动画时长（秒）
Use Local Space是否使用本地坐标（建议 UI 元素开启）
Rotation ModeShortestPath（最短路径）或 FullRotation（完整旋转）
Init Behaviour组件启用时自动定位到起点/终点/不处理

Edit 模式预览

在 调试 标签页中，无需进入 Play 模式即可直接预览动画效果，方便在编辑场景时快速验证动画参数。

  • 模拟预览 — 预览不会真正修改物体的原始 Transform 状态，仅作视觉模拟
  • 自动还原 — 进入 Play 模式或点击选中其他物体时，物体会立即回到预览前的原始状态
  • 完整控制 — 支持播放到 Display / Origin、立即跳转、Toggle 切换、停止还原等操作
  • 进度显示 — 面板中实时展示进度条和当前动画状态

Transform 模式下插值 Position、Rotation、Scale 及 RectTransform 属性；ImageFill 模式下直接驱动 fillAmount 值变化，预览结束自动还原。

代码调用速查

// 获取引用 var anim = GetComponent<SimpleAnimator>(); // 基本播放 anim.ToDisplay(); // 到终点 anim.ToOrigin(); // 到起点 anim.Toggle(); // 自动判断方向 // 立即跳转（无动画） anim.SetDisplayImmediate(); anim.SetOriginImmediate(); // 重复播放（来回 N 次） anim.ToDisplay(3); // 到终点再回来，重复 3 次 // 停止 anim.KillAll();

路径动画器

PathAnimator — 简易动画器的进阶版

它能做什么

快速、一键、无代码地生成物体或 UI 的路径动画，让目标对象沿一组路径点平滑移动。可以理解为简易动画器的进阶版本。适用于：巡逻路线、相机轨道、UI 引导动画、物体传送带等。

使用步骤

1

右键创建：GameObject > 模板 > 动画 > 路径动画器

2

在 Inspector 中点击 QuickInit，会自动生成 3 个路径点子物体。

3

在 Scene 视图中拖动路径点位置。需要更多点可点击 Add Waypoint。

路径会以 Gizmo 线条实时显示在 Scene 视图中。

4

跳转到下方获取完整 API，或进入调试标签页直接预览测试效果。

核心配置

路径插值 — Interpolation

Linear — 直线连接路径点，拐角处方向突变。适合机械轨道、传送带。

CatmullRom — Catmull-Rom 样条曲线，平滑通过所有路径点。适合相机轨道、飞行路线。

循环方式 — Loop Mode

Once — 播放一次后停止。 Loop — 无限循环。 PingPong — 到达终点后反向，往复运动。

运动朝向 — Look At Mode

None — 不控制旋转。 Forward — 始终面朝移动方向。 Fixed — 看向固定目标。 Waypoint — 按路径点旋转插值（Linear→Slerp，CatmullRom→Squad）。

速度模式 — Speed Mode

Constant — 匀速，每段时间与段长成正比。 UniformSegment — 每段等时，长段快短段慢。

其他属性

Duration — 全程时长（秒）。 Close Path — 首尾闭合。 Use Local Space — 本地坐标系计算。 Init Behaviour — 启动时定位。 Show Path Gizmo — Scene 视图路径预览线。

Edit 模式预览

在 调试 标签页中，无需进入 Play 模式即可预览完整路径动画，方便调参和验证路径效果。

  • 模拟预览 — 物体沿路径移动仅是视觉模拟，预览结束后自动还原到原始位置，不会改变物体的实际 Transform 状态
  • 自动还原 — 进入 Play 模式或点击选中其他物体时，物体会立即回到预览前的原始状态，无需手动撤销
  • 完整路径支持 — 正向/反向播放、暂停/继续、停止及回到起点，支持 Linear / CatmullRom 插值、Once / Loop / PingPong 循环模式、Forward / Fixed 朝向模式
  • 实时进度 — 面板中显示当前播放进度条和状态文字

代码调用速查

var path = GetComponent<PathAnimator>(); path.PlayForward(); path.PlayReverse(); path.Pause(); path.Resume(); path.Stop(); path.StopAndReset(); // 停止并回到第一个点 // 直接跳到某个点 path.SnapToPoint(2); // 跳到第 3 个路径点 path.SnapToProgress(0.5f); // 跳到路径 50% 位置 // 只计算位置（不移动物体） Vector3 pos = path.EvaluatePathAtProgress(0.75f);

帧动画控制器

LegacyAnimationController — 精确控制 Legacy Animation 的播放帧段

它能做什么

对挂载了 Legacy Animation 组件的物体，按「起始帧 ~ 结束帧」精确播放动画片段。适用于：机械运动、开关门、设备状态切换等需要帧级控制的场景。

GIF: 一个门从关闭播放到打开的帧动画演示展示 Play / SnapToFrame 的效果

使用步骤

1

右键创建：GameObject > 模板 > 动画 > 帧动画控制器

2

确保目标物体上有 Animation（Legacy）组件并挂载了动画片段。

3

在 Inspector 中点击 QuickInit，自动读取所有可用的动画片段到列表中。

4

跳转到下方获取完整 API，或进入调试标签页直接测试效果。

代码调用速查

var ctrl = GetComponent<LegacyAnimationController>(); // 播放当前选中的片段（返回播放时长） float duration = ctrl.Play(); // 播放指定片段并自定义帧范围 ctrl.Play("Open", startFrame: 0, endFrame: 30, speed: 1.5f); // 跳到某一帧 ctrl.SnapToFrame(15); ctrl.SnapToStartFrame(); ctrl.SnapToEndFrame(); // 暂停 / 恢复 ctrl.Pause(); ctrl.Resume();

相机模块

三种相机控制模式，覆盖大部分 3D 场景需求

内置三种模式

  • 鸟瞰式移动相机 — 俯视角拖拽平移 + 滚轮调高度，适合地图/沙盘类
  • 旋转缩放式相机 — 围绕目标旋转 + 缩放，适合展示/查看模型
  • 第一人称飞行移动 — WASD + 鼠标，适合场景漫游

三种模式通过 CameraController 统一管理，运行时可随时切换。

GIF: 三种相机模式的对比演示可以做成三个并排小 GIF 或一个切换演示

创建方式

右键 Hierarchy > GameObject > 模板 > 相机 > 相机控制器

创建后在 Inspector 的 Camera Type 下拉框选择模式。

相机控制器

CameraController — 核心组件，管理相机状态切换

使用步骤

1

创建后它会自动查找场景中的 Main Camera。也可手动拖拽指定 Target Camera。

2

选择 Camera Type：鸟瞰式 / 旋转缩放式 / 第一人称飞行。

3

展开对应模式的折叠栏配置参数，进入调试标签页可直接测试相机操控效果。

鸟瞰式相机 — 操作方式

操作功能
鼠标左键拖拽平移视野（水平方向移动）
鼠标中键拖拽旋转视角（俯仰 + 偏航）
滚轮滚动升降相机高度

旋转缩放式相机 — 操作方式

操作功能
鼠标左键拖拽围绕目标物体旋转（俯仰 + 偏航）
滚轮滚动拉近 / 拉远与目标之间的距离
松手后惯性旋转自动衰减滑动，无需手动减速

第一人称飞行 — 操作方式

按键功能
W / ↑前进
S / ↓后退
A / ←向左平移
D / →向右平移
E上升
Q下降
=加速移动

代码切换模式

// 获取单例 var cam = CameraController.Instance; // 切换模式 cam.SetCameraType(CameraController.CameraTypeEnum.旋转缩放式相机); // 聚焦到某个物体 cam.DoFocus(targetTransform, relativeEulers, relativePosition);

视角切换器

CameraZoomController — 预设多个机位，一键切换相机视角

它能做什么

提前设置好场景中几个关键观察位置（机位），运行时一键即可让相机平滑飞过去。适用于：建筑楼层切换、展品多角度查看、场景导览跳转等。

GIF: 点击按钮在不同机位之间平滑切换展示相机飞到不同预设位置和角度的动画过程

使用步骤

1

创建：GameObject > 模板 > 相机 > 视角切换器

2

在 Inspector 的 Informs 列表中添加机位配置，至少填入 Name 和 Target。

3

如果当前由 CameraController 管理的是旋转缩放式相机，重点配置 Zoom Distance、Rotation Angle X/Y。

4

如果当前是其他相机模式，则实际使用的是 Camera Offset 和 Look Eulers 来决定机位和朝向；Zoom Distance / Rotation Angle 在这类模式下不是主要控制项。

5

进入调试标签页可直接代码调用或手动触发切换，实时测试机位切换效果。

Informs 字段说明

字段说明
Name机位名称，供代码按名称切换
Target观察目标 Transform
Zoom Distance旋转缩放式相机下的目标距离
Rotation Angle Y / X旋转缩放式相机下的目标偏航角 / 俯仰角
Duration切换动画时长
Camera Offset非旋转缩放模式下，相机相对目标的位移偏移
Look Eulers非旋转缩放模式下的固定朝向；留空向量时自动朝向目标

代码调用

var zoom = CameraZoomController.Instance; // 按名称切换到预设机位 zoom.SetCameraTarget("Lobby"); // 直接传配置对象 zoom.SetCameraTarget(new CameraZoomController.CameraCenterTargetInform { name = "MachineA", target = targetTransform, zoomDistance = 8f, rotationAngleY = 35f, rotationAngleX = 20f, duration = 1.2f }); // 仅旋转缩放式相机可用：绕目标旋转 zoom.RotateAroundTarget("Lobby", cycleDuration: 5f); // 停止当前镜头动画 zoom.StopCurrentAnimation();

视口偏移控制器

CameraViewportController — 画面焦点左右偏移

它能做什么

当侧边面板展开时，通过偏移相机镜头让主画面焦点不被遮挡。效果类似地图应用打开详情面板时画面中心的偏移。

GIF: 面板展开时画面焦点自动偏移的效果对比有偏移和无偏移时的视觉差异

使用步骤

1

创建：GameObject > 模板 > 相机 > 视口偏移控制器

2

在 Inspector 中设置 Shift Amount 作为左右偏移量，设置 Transition Duration 控制过渡时长。

3

如果希望场景启动后默认偏到左侧，可开启 Init Set Left。

4

通常把它和侧边栏开关、详情面板展开按钮联动：面板打开时 SetToLeft()，关闭时 SetToOri()。

关键配置

属性说明
Transition Duration偏移动画时长
Shift Amount镜头左右偏移量，范围 0~1
Ease Curve偏移动画缓动曲线

Enable Transition是否立即执行命令；关闭后会暂存命令，重新开启时再执行

代码调用

var vpc = CameraViewportController.Instance; vpc.SetToLeft(); // 画面偏左（焦点让给右侧面板） vpc.SetToRight(); // 画面偏右 vpc.SetToOri(); // 回到居中

网络通信模块

HTTP 请求和 WebSocket 长连接的开箱即用方案

模块包含

  • HTTP 数据源 — 单个 HTTP 请求组件，支持 GET / POST / PUT / DELETE / PATCH / HEAD / OPTIONS
  • HTTP 数据源管理器 — 统一管理多个 API 接口，自带对象池和缓存
  • WebSocket 客户端 — 长连接，自动重连 + 心跳

HTTP 数据源

HttpDataSource — 直接在 Inspector 中配置 HTTP 请求

它能做什么

不写代码也能发 HTTP 请求。URL、方法、参数、请求头都可以在 Inspector 中配置；如果运行时需要临时改接口、参数或返回类型，也可以用 HttpRequestOptions 把 url / method / query / headers / body / responseType 明确写出来。

使用步骤

1

创建：GameObject > 模板 > 网络通信 > HTTP 数据源

2

在 Inspector 的请求配置页中配置：

  • 填写 URL
  • 选择 Method（GET / POST / PUT / DELETE / PATCH / HEAD / OPTIONS）和 Response Type（Text / Json / Binary / Image）
  • 如果需要请求体，设置 Body Type 和内容；GET / HEAD 一般不带 Body，只通过 Query 传参
  • 如需要，添加 Parameters、Headers，并选择是否使用 Auth Provider 自动补充鉴权 Header
  • 同一页底部即可绑定 OnResponseReceived / OnErrorReceived / OnImageReceived
3

在调试标签页手动发送请求测试、查看历史响应；如果需要额外打印运行日志，再开启 输出到 Console。

4

跳转到下方查看按场景拆分的调用方式。

支持范围

  • 请求方法：GET / POST / PUT / DELETE / PATCH / HEAD / OPTIONS
  • 响应类型：Text / Json / Binary / Image
  • 请求体：字符串、字节数组、本地文件
  • 参数入口：URL Query、JSON Body、application/x-www-form-urlencoded、multipart/form-data
  • 鉴权：自定义 Header 或 IAuthHeaderProvider 自动注入

下面的案例按真实支持能力拆成折叠项，覆盖常规网页前端里最常见的请求方式。先看场景再展开代码会更直观。

代码案例库

读取列表、详情或 JSON 对象

适用场景：页面初始加载、详情读取、基础 GET 查询、直接反序列化 JSON。

var listOptions = new HttpRequestOptions("https://api.example.com/users", HttpMethod.GET) { responseType = HttpResponseType.Json } .AddQuery("page", "1") .AddQuery("pageSize", "20"); http.SendRequest(listOptions, response => { if (!response.IsSuccess) { Debug.LogError(response.Error); return; } var model = response.FromJson<UserListResponse>(); Debug.Log(model.total); }); var detailOptions = new HttpRequestOptions("https://api.example.com/users/42", HttpMethod.GET) .AddQuery("page", "1") .AddQuery("pageSize", "20"); http.SendRequest(detailOptions, response => { if (!response.IsSuccess) { Debug.LogError(response.Error); return; } Debug.Log(response.Text); });

关键点：GET / HEAD 不带请求体；需要直接反序列化时可把 responseType 设成 Json，再用 FromJson<T>()。

提交 JSON 表单

适用场景：登录、保存设置、创建记录、更新表单。

var saveOptions = new HttpRequestOptions("https://api.example.com/profile", HttpMethod.PUT) .AddHeader("Content-Type", "application/json") .SetJsonBody("{\"nickname\":\"Alice\",\"age\":18}"); http.SendRequest(saveOptions, response => { if (!response.IsSuccess) { Debug.LogError(response.Error); return; } Debug.Log(response.Text); });

关键点：JSON 提交最常用的是 SetJsonBody；方法通常是 POST / PUT / PATCH。

普通表单提交 (x-www-form-urlencoded)

适用场景：传统登录接口、OAuth token 交换、后端仍按表单字段接收的旧接口。

var formFields = new List<FormParameter> { new FormParameter("username", "alice"), new FormParameter("password", "123456"), new FormParameter("remember", "true", ParameterType.Bool) }; var formOptions = new HttpRequestOptions("https://api.example.com/login", HttpMethod.POST) .SetFormUrlEncodedBody(formFields); http.SendRequest(formOptions, response => { Debug.Log(response.IsSuccess ? response.Text : response.Error); });

关键点：这会生成真正的 application/x-www-form-urlencoded 请求体，不再只是 Query 或 JSON。

发送纯文本或自定义 Content-Type

适用场景：发纯文本、XML、日志内容，或者接口要求非 JSON 的字符串请求体。

var textOptions = new HttpRequestOptions("https://api.example.com/raw", HttpMethod.POST) .SetTextBody("plain-text-payload", "text/plain"); http.SendRequest(textOptions, response => { Debug.Log(response.IsSuccess ? response.Text : response.Error); });

关键点：SetTextBody 也是字符串请求体，只是 Content-Type 不再固定为 application/json。

用参数自动组装请求

适用场景：接口字段简单，想继续用组件里的 Parameters 列表管理输入。

var parameters = new List<FormParameter> { new FormParameter("keyword", "alice"), new FormParameter("page", "1", ParameterType.Int), new FormParameter("needTotal", "true", ParameterType.Bool) }; http.SetFormParameters(parameters); http.SendRequest();

关键点：组件模式下，GET 会把参数拼到 URL；POST / PUT / PATCH 且未手写 Body 时，可在 Inspector 里切成 JSON Body。

自定义 Header 与 Token

适用场景：鉴权、语言区分、设备标识、灰度环境标记。

var authOptions = new HttpRequestOptions("https://api.example.com/orders", HttpMethod.GET) .AddHeader("Accept-Language", "zh-CN") .AddHeader("X-Device-Id", SystemInfo.deviceUniqueIdentifier); authOptions.includeAuthHeaders = true; http.SendRequest(authOptions, response => { if (response.IsSuccess) Debug.Log(response.Text); });

关键点：includeAuthHeaders = true 时，会从同物体或父物体上的 IAuthHeaderProvider 自动补鉴权 Header。

DELETE / PATCH / HEAD / OPTIONS

适用场景：删除记录、局部更新、探测接口是否存在、检查服务器支持能力。

var deleteOptions = new HttpRequestOptions("https://api.example.com/users/42", HttpMethod.DELETE); var patchOptions = new HttpRequestOptions("https://api.example.com/users/42", HttpMethod.PATCH) .SetJsonBody("{\"status\":\"disabled\"}"); var headOptions = new HttpRequestOptions("https://api.example.com/users/42", HttpMethod.HEAD); var optionsRequest = new HttpRequestOptions("https://api.example.com/users", HttpMethod.OPTIONS); http.SendRequest(deleteOptions, response => { Debug.Log(response.IsSuccess ? "Deleted" : response.Error); }); http.SendRequest(patchOptions, response => { Debug.Log(response.IsSuccess ? response.Text : response.Error); });

关键点：HEAD 和 OPTIONS 通常只看 StatusCode / Headers；不要给 GET / HEAD 带请求体。

统一处理错误与状态码

适用场景：前端页面需要把网络错误、业务失败、空响应分开处理。

http.SendRequest(options, response => { if (response == null) { Debug.LogError("No response object."); return; } if (!response.IsSuccess) { Debug.LogError($"HTTP {response.StatusCode}: {response.Error}"); return; } if (string.IsNullOrEmpty(response.Text)) { Debug.LogWarning("Response text is empty."); return; } Debug.Log(response.Text); });

关键点：不要只判断 Text 是否为空；应优先用 IsSuccess、StatusCode、Error 分支处理。

图片响应

适用场景：头像、海报、截图、运行时纹理下载。

var imageOptions = new HttpRequestOptions("https://picsum.photos/200/300", HttpMethod.GET) { responseType = HttpResponseType.Image }; http.SendRequest(imageOptions, response => { if (!response.IsSuccess || response.Texture == null) return; rawImage.texture = response.Texture; }); http.OnImageDelegate += texture => { rawImage.texture = texture; };

关键点：图片响应走 response.Texture；如果想用 UnityEvent / Delegate，也可以直接绑定图片事件。

二进制响应

适用场景：文件下载、音频字节流、自定义二进制协议包。

var binaryOptions = new HttpRequestOptions("https://api.example.com/report.bin", HttpMethod.GET) { responseType = HttpResponseType.Binary }; http.SendRequest(binaryOptions, response => { if (!response.IsSuccess || response.Bytes == null) return; Debug.Log($"Downloaded {response.Bytes.Length} bytes"); File.WriteAllBytes(Path.Combine(Application.persistentDataPath, "report.bin"), response.Bytes); });

关键点：Binary 不会触发字符串 UnityEvent，处理入口是 response.Bytes。

上传字节数组或本地文件

适用场景：上传导出文件、截图、日志包或本地资源。

// 方案 1：运行时上传字节数组 byte[] screenshotBytes = File.ReadAllBytes(imagePath); var uploadBytesOptions = new HttpRequestOptions("https://api.example.com/upload", HttpMethod.POST) .SetBytesBody(screenshotBytes, "image/png"); http.SendRequest(uploadBytesOptions, response => { Debug.Log(response.IsSuccess ? "Upload ok" : response.Error); }); // 方案 2：直接上传本地文件 var fileOptions = new HttpRequestOptions("https://api.example.com/upload", HttpMethod.POST) .SetLocalFileBody(logZipPath, "application/zip"); http.SendRequest(fileOptions, response => { Debug.Log(response.IsSuccess ? response.Text : response.Error); });

关键点：字节流和本地文件都走请求体，不通过 Parameters 列表。

multipart 表单上传

适用场景：上传头像、截图、附件，同时还要带描述、业务 ID、分类字段。

var multipartFields = new List<FormParameter> { new FormParameter("userId", "42", ParameterType.Int), new FormParameter("category", "avatar") }; var multipartFiles = new List<MultipartFormFileParameter> { new MultipartFormFileParameter("file", avatarPath, "avatar.png", "image/png") }; var multipartOptions = new HttpRequestOptions("https://api.example.com/upload/avatar", HttpMethod.POST) .SetMultipartFormDataBody(multipartFields, multipartFiles); http.SendRequest(multipartOptions, response => { Debug.Log(response.IsSuccess ? response.Text : response.Error); });
// 运行时直接传内存字节 var bytes = File.ReadAllBytes(avatarPath); var runtimeMultipart = new HttpRequestOptions("https://api.example.com/upload/avatar", HttpMethod.POST) .AddFormField("userId", "42", ParameterType.Int) .AddMultipartFile("file", bytes, "avatar.png", "image/png"); http.SendRequest(runtimeMultipart, response => { Debug.Log(response.IsSuccess ? response.Text : response.Error); });

关键点：需要字段和文件混发时，用 SetMultipartFormDataBody 或 AddMultipartFile；如果走管理器，可改用 Request(key, HttpRequestOptions, callback) 传入同样的 options。

临时覆盖 Inspector 默认配置

适用场景：场景里已经挂好默认 URL / Header / 参数规则，但某次请求需要临时改地址、方法或返回类型。

var options = http.BuildConfiguredOptions(); options.url = "https://api.example.com/users/42"; options.method = HttpMethod.DELETE; options.responseType = HttpResponseType.Json; options.headers["X-Trace-Id"] = Guid.NewGuid().ToString("N"); http.SendRequest(options, response => { Debug.Log(response.IsSuccess ? response.Text : response.Error); });

关键点：BuildConfiguredOptions() 会先继承 Inspector 当前配置，再按代码覆盖。

多接口项目交给管理器

适用场景：同一页面或系统里有多个 API，需要统一 Base URL、缓存和对象池。

var mgr = GetComponent<HttpDataSourceManager>(); mgr.Request("getUserInfo", new List<FormParameter> { new FormParameter("userId", "42", ParameterType.Int) }, text => { Debug.Log(text); }); var overrideOptions = new HttpRequestOptions { method = HttpMethod.PATCH, responseType = HttpResponseType.Json }.SetJsonBody("{\"status\":\"disabled\"}"); mgr.Request("updateUserStatus", overrideOptions, response => { Debug.Log(response.IsSuccess ? response.Text : response.Error); });

关键点：`HttpDataSource` 适合单个独立请求；接口数量一多，`HttpDataSourceManager` 的对象池和缓存会更省心。

当前边界

这一版 HTTP API 已覆盖大多数网页前端常用场景，包括 Query、JSON、application/x-www-form-urlencoded 和 multipart/form-data。当前仍未覆盖的是流式响应处理、cookie jar 和浏览器级跨域语义；如果继续向浏览器请求栈靠拢，优先级最高的是流式响应。

关于鉴权

如果请求需要 Token，在同一物体或父物体上挂载 BearerTokenProvider 组件，填入 Token，并在 HTTP 数据源的请求头区块保持 使用 Auth Provider 开启。需要图片返回时，在 Inspector 中把 Response Type 设为 Image，代码里则设置 responseType = HttpResponseType.Image。

怎么理解这些配置

如果你只想尽快把下拉框用起来，最常改的其实就三类：

  • 内容怎么来 — 直接在 Options 里填文本，或者运行时用 SetOptions / AddOption 动态改。
  • 展开后长什么样 — 想让列表往上弹还是往下弹、是一次性全展开，还是只显示几项并自动滚动，都在行为配置里控制。
  • 外观走颜色还是图片 — 简单场景只配颜色就够用；如果要做更明确的视觉状态，再补按钮、列表和选项的 Sprite。

颜色模式怎么用

颜色模式适合大多数普通表单、设置面板和工具栏场景。你只需要关心：

  • 按钮背景色 — 收起状态下主按钮是什么颜色。
  • 列表背景色 — 展开后整块下拉面板是什么颜色。
  • 选项默认 / 悬停 / 选中颜色 — 鼠标移入、点击选中时怎么变化。
  • 文字默认 / 选中颜色 — 让选中项更突出，或者保证深色背景下可读。

如果你不想处理图片资源，直接把这几组颜色调顺，基本就能得到一套完整可用的下拉框外观。

精灵模式怎么用

  • 按钮和列表 仍然是单张背景图覆盖，适合做整体面板皮肤。
  • 选项背景 需要一次性补齐 默认 / 悬停 / 选中 三张状态图，这样鼠标移入和选中时才会正确切换。
  • 安全回退 — 如果三张图没配全，组件不会半吊子地只切一部分状态，而是自动退回颜色模式，继续使用默认色、悬停色和选中色。

可以把它理解成“颜色模式永远可用，精灵模式是完整配置后的增强版”。这样团队里即使有人只先补了一张图，也不会把交互状态弄乱。

Effect2UGUI

设计图 + UI 元素目录，生成 uGUI 预制体粗稿

它能做什么

这是一个基于设计图和 UI 元素，让 AI 自动生成预制体的功能。

可以把它看成一个 Skill。区别是它集成在 Unity 里，不在 Codex 或 Claude Code 里。因为在 Unity 里指定资源、生成 prefab 更方便。

简单说，就是把一套任务流程固定下来，然后让 AI Agent 按这个流程一步一步做。

环境需求

本功能要求 Code Agent 支持图片输入，并具备基础 OCR / 视觉理解能力。仅文本模型不建议使用；否则只能根据资源名、尺寸和人工说明推断布局，生成精度不可控。
